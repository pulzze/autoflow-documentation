# switch

## Summary

This Action returns values that fits on switch case.<br/>
If the input value is not in the case list, it returns the value of default.

## Description

The <span class="action-field">SWITCH</span> ACTION evaluates an expression, matching the expression's value against a series of <span class="action-field">*CASE*</span> clauses,
and executes statements after the first <span class="action-field">*CASE*</span> clause with a matching value, until <span class="action-field">*DEFAULT*</span> statement is encountered. <span class="action-field">*DEFAULT*</span> clause of a switch statement will be jumped to if no <span class="action-field">*CASE*</span>  matches the expression's value.

## Fields

In <b>*SWITCH Action*</b> there are 5 type fields.

- <span class="action-field">SWITCH</span> : Criteria for SWITCH
- <span class="action-field">CASE</span> : Target's CASE
- <span class="action-field">Return</span> : Return result IF the Creteris is 'CASE'
- <span class="action-field">Add CASE</span> : Add more CASE
- <span class="action-field">Default Return</span> : Default return result

### - <span class="action-field">SWITCH</span>

This field is Criteria for <span class="action-field">SWITCH</span>.

:::tip
But, I suggest use <b>*'Reference Data'*</b> or <b>*'Action'*.</b> type. <br/>
Because we need value that changes through the case.
:::

### - <span class="action-field">CASE</span>

This field is Target's <span class="action-field">CASE</span>.<br/>
Usually, use <b>*String*</b> Type, but you can use another types,<br/>
For examples, <b>*Object, Referece Data, Action...*</b>

<img src={switch03} alt="Version Location" class="myResponsiveImg" width="800px"/>

:::tip
But, I suggest use kind of Fixed Value. <br/>
Such as <b>*'string'*</b> or <b>*'number'*.</b> type. <br/>
Because Case requires a fixed value.
:::

### - <span class="action-field">Return</span>

This field is 'Return' result If the <span class="action-field">SWITCH</span> is same with <span class="action-field">CASE</span>.<br/>
You can change <span class="action-field">Return</span> Type to <b>*Object, Referece Data, Action...*</b>

<img src={switch04} alt="Version Location" class="myResponsiveImg" width="800px"/>

### - <span class="action-field">Add CASE</span>

If you want to make more <span class="action-field">CASE</span>, <br/>
Click <span class="action-field">Add CASE</span> button.<br/>
Then you can make more <span class="action-field">CASE</span> whatever you want!

<img src={switch05} alt="Version Location" class="myResponsiveImg" width="800px"/>

### - <span class="action-field">Default Return</span>

This field is <b>*Default Return*</b><br/>

## Example

Let's Use SWITCH ACTION!
### Target - *String* Example

Let's imagine.<br/>
I want to make an if statement true only when data with reference data is data that meets the criteria I set.

:::info goal
We will put the string ```'Autoflow'``` in the ```ref_data```,<br/>
If the value is the same as <span class="action-field">CASE</span> criterion, the value will be returned as ```'ref is autoflow!'```. If not, I would return ```Default Return``` value.

We will get Action result like below.<br/>
- If ```ref_data``` is the same as <span class="action-field">CASE</span>,
    - it would return ```'ref is Autoflow!'```
- If ```ref_data``` is not the same as <span class="action-field">CASE</span>,
    - it would return ```'ref is not in CASE!'```
:::
