# try

## Summary

This Action return values which catch error result or onError result.

## Description

The <span class="action-field">try</span> ACTION is comprised of <span class="action-field">value</span> field and <span class="action-field">onError</span> field.

## Fields

In <b>*TRY Action*</b> there are 2 type fields.

- <span class="action-field">value</span> : Target to accept Error
- <span class="action-field">onError</span> : If the <b>value</b> is on Error, Try Action will do.

<img src={try01} alt="Version Location" class="myResponsiveImg" width="800px"/>
<hr/>

### - <span class="action-field">value</span>

This field is where the Action catches the error with try when the error is likely to occur.<br/>
I suggest using the Action for that field. This is because the fixed value(```string```, ```array```...) is not subject to errors.

<img src={try02} alt="Version Location" class="myResponsiveImg" width="800px"/>

### - <span class="action-field">onError</span>

If an error occurs in <span class="action-field">value</span>, the Try action can determine what to return.<br/>
You can select any types, such as <b>*'string'*</b>, <b>*'number'*.</b> or <b>*'Action'*</b> type. <br/>

<img src={try03} alt="Version Location" class="myResponsiveImg" width="800px"/>
